Lenguajes de Marcas

Temas y conceptos clave sobre lenguajes de marcas

¿Qué es un lenguaje de marcas?

Un lenguaje de marcas es un sistema de codificación de documentos en el que, junto con el texto, se incluyen etiquetas o marcas que añaden información sobre la estructura del contenido o su presentación. Por ejemplo, en HTML las etiquetas <p>, <h1>, <img>, etc., indican la estructura y formato del texto mostrado. A diferencia de los lenguajes de programación, los lenguajes de marcas no tienen variables ni operaciones aritméticas; su función es describir o presentar el contenido, no ejecutarlo. El uso histórico de lenguajes de marcas proviene del ámbito editorial (entre autores y editores). Los lenguajes de marcas se clasifican según su propósito principal. Por ejemplo, en el pasado se distinguían entre marcado de presentación (enfocado al formato visual del texto) y marcado descriptivo (etiquetas semánticas que describen el contenido sin dictar su aspecto). Hoy en día, una clasificación más común agrupa los lenguajes en categorías como presentacionales, estructurales o descriptivos, documentales/científicos y normativos. En términos generales:

Indican directamente el formato de cada fragmento de texto (por ejemplo, centrar, negrita, color). Eran fáciles de usar para documentos pequeños, pero poco flexibles para procesar información automáticamente. HTML (originalmente) entraría aquí al usar etiquetas como <b> o atributos de estilo.

Etiquetan las partes del documento según su significado o estructura lógica. Por ejemplo, <title> indica un título, pero el navegador decide el estilo. XML y sus derivados son ejemplos paradigmas de marcado descriptivo.

Lenguajes orientados a redactar documentos técnicos o académicos complejos, enfocándose en la semántica del contenido. LaTeX y DocBook son ejemplos destacados.

Lenguajes que definen la estructura de otros lenguajes o documentos. Incluyen DTD, XSD y SGML como metalenguaje base.

Clasificación de lenguajes de marcas

Lenguajes de marcas de presentación

  • HTML: Los lenguajes de marcas de presentación se centran en el formato visual del texto. El más conocido es HTML (HyperText Markup Language), el estándar básico para construir páginas web. HTML permite estructurar una página indicando encabezados (<h1>–<h6>), párrafos (<p>), listas, tablas, enlaces, imágenes, etc.

    “HTML es el código que se utiliza para estructurar y desplegar una página web y sus contenidos”.

    Es importante resaltar que HTML no es un lenguaje de programación, sino un lenguaje de marcado: sus etiquetas sirven para encerrar fragmentos de contenido y aplicarles un comportamiento o estilo concreto. 
    Por ejemplo, un documento HTML5 básico incluye al inicio la declaración 
  <!DOCTYPE html> y las etiquetas <html>, <head> y <body>. 
    

    
Dentro de <head> se define el título y metadatos, mientras que <body> contiene el contenido visible. Un ejemplo mínimo sería:

    
<!DOCTYPE html>
<html>
<head>
<title>Título de la página</title>
</head>
<body>
<h1>Encabezado principal</h1>
<p>Este es un párrafo.</p>
</body>
</html>
  • XHTML: XHTML es HTML definido como aplicación XML: respeta todas las reglas de sintaxis de XML. Por ello, XHTML es más estricto: todos los elementos deben estar bien anidados, en minúsculas, con atributos entre comillas y debidamente cerrados (incluso las etiquetas vacías, p.ej. <br/>). Por ejemplo, en XHTML el elemento raíz <html> debe llevar el atributo de espacio de nombres xmlns y cerrarse apropiadamente. La principal ventaja es facilitar la interoperabilidad con otras tecnologías XML y el procesamiento riguroso de los documentos; la desventaja es que los navegadores originales forzaban un comportamiento más tolerante que a veces ignoraba errores en HTML puro.
    Aunque XHTML fue promovido por el W3C en los 2000s, con la llegada de HTML5 su uso se ha reducido, y hoy la mayoría de sitios emplean HTML5 clásico para evitar los requisitos estrictos de XML. Sin embargo, XHTML sirvió como ejercicio para “endurecer” las buenas prácticas de HTML (cerrar etiquetas, usar minúsculas, etc.).

Lenguajes de marcas descriptivos o estructurales

  • XML: El más importante de esta categoría es XML (Extensible Markup Language).

    “XML es un lenguaje de marcado similar a HTML” que significa Extensible Markup Language y es una especificación del W3C para un propósito general.

    A diferencia de HTML, XML no tiene etiquetas predefinidas; el desarrollador define sus propias etiquetas según la aplicación. Su propósito principal es el intercambio de datos entre distintos sistemas (p.ej. aplicaciones web, servicios, bases de datos) de forma estructurada.
    Un documento XML debe ser bien formado: cada etiqueta abierta debe cerrarse, respetar anidación correcta y tener un único elemento raíz. Además, puede validarse contra un esquema (DTD o XSD) para comprobar su estructura. XML es ampliamente usado en servicios web (SOAP), sindicaciones (RSS), configuración de aplicaciones, y como base para otros lenguajes (por ejemplo, XHTML, MathML para matemáticas, SVG para gráficos vectoriales, XUL, etc.). Un ejemplo sencillo de XML podría ser:

    En este fragmento, <persona>, <nombre>, <edad>, <email> son etiquetas definidas por el autor para describir los datos de una persona.

    • YAML/JSON

  • YAML: En la categoría descriptiva se incluyen también YAML y JSON, que, aunque no son estrictamente “lenguajes de marcas” tradicionales (no usan etiquetas angulares), se usan para serializar datos de forma estructurada y legible.

    Es un formato de serialización de datos legible por humanos, diseñado para configurar aplicaciones y describir estructuras complejas con una sintaxis clara basada en indentación.

    “YAML es un lenguaje versátil y legible por humanos para serializar datos, que se utiliza a menudo para escribir archivos de configuración”. 
    Por ejemplo, una lista en YAML puede escribirse usando guiones y sangrías:
        
    
- fruta: Manzana
  color: Rojo
- fruta: Plátano
  color: Amarillo
    YAML admite colecciones (listas, diccionarios) y escalables (números, cadenas, booleanos). Su principal ventaja es la simplicidad y legibilidad para los humanos, aunque su sintaxis sensible a la indentación exige cuidado.

  • JSON: (JavaScript Object Notation) es un formato ligero de intercambio de datos basado en texto. Según IBM, “JSON es un formato ligero de intercambio de datos” que resulta fácil de leer y escribir tanto para humanos como para máquinas. Utiliza convenciones similares a los lenguajes de la familia C (clave-valor entre llaves y listas entre corchetes).



    JSON es muy popular en aplicaciones web modernas (APIs REST, configuración, almacenamiento en bases de datos NoSQL) por su simplicidad y compatibilidad. Aunque su origen es JavaScript, hoy es independiente de cualquier lenguaje de programació. JSON es más estricto en sintaxis que YAML (p. ej. obliga a comillas en cadenas) pero también más fácil de procesar automáticamente.

  • GML: GML es un sublenguaje de XML diseñado por el Consorcio Geoespacial Abierto (OGC) para representar información geográfica. Se define como una gramática en esquema XML para modelar características geográficas (puntos, líneas, polígonos, atributos) en forma legible. GML permite intercambiar datos GIS (Sistemas de Información Geográfica) entre programas, y se usa en servicios web de mapas y bases de datos geoespaciales. Su sintaxis es XML con etiquetas como <gml:Point>, <gml:coordinates>, etc., definidas por el estándar OGC.


    • GML es especialmente útil para representar mapas, datos satelitales y cualquier información que requiera precisión geográfica.

Lenguajes de marcas documentales o científicos

Los lenguajes de marcas documentales o científicos se diseñan para la creación de documentos complejos, con notación especial para fórmulas, referencias, tablas de contenido, bibliografía, etc.

  • LaTeX: Un representante emblemático es LaTeX, un sistema de preparación de documentos basado en el software TeX de Donald Knuth. LaTeX facilita la escritura de documentos académicos o técnicos de alta calidad.

    “LaTeX es un sistema de preparación de documentos que facilita la escritura de documentos científicos y técnicos”, permitiendo expresar con facilidad fórmulas matemáticas complejas, gestionar referencias y formatear el documento de forma automática.

    A diferencia de los procesadores de texto WYSIWYG (como Word), en LaTeX el autor escribe el contenido junto con comandos de formato en texto plano (p. ej. \section{} para secciones, \begin{equation} para ecuaciones). Tras compilar el archivo .tex, LaTeX produce salida PDF/PS de alta calidad tipográfica. Por ejemplo, el código \(\int_0^1 x^2 dx\) genera la integral matemática correspondiente. LaTeX separa claramente contenido (texto, ecuaciones) de formato (estilo de capítulo, numeración de ecuaciones, índices), lo cual es ventajoso en documentos extensos. Hoy en día LaTeX es el estándar de facto en publicaciones científicas de física, matemáticas, ingeniería, y ciencias sociales (tesis, artículos de revista).

  • DocBook: DocBook se originó como una DTD SGML y luego versión XML destinada a documentar software, manuales técnicos y libros.

    “DocBook es una aplicación del estándar SGML/XML e incluye una DTD propia que se utiliza de manera más destacada en el área de la documentación técnica”.

    En la versión actual (DocBook 5) está formalmente definido por un esquema RELAX NG con reglas Schematron para validación. DocBook es semántico: en lugar de indicar directamente el estilo, marca la estructura del documento (libro, capítulo, sección, párrafo, listados de código, etc.) de forma neutra. 
    
    Por ejemplo, un capítulo se describe con
    <chapter><title>Título</title>...contenido...</chapter> 
    Sin decir si será impreso en negrita o itálica. Luego, hojas de estilo XSL (o procesadores especializados) transforman el DocBook a HTML, PDF, manpages, EPUB y
    otros formatos. 
    Este enfoque permite escribir la documentación una sola vez y publicarla en múltiples formatos. 
    DocBook cuenta con un comité técnico (OASIS) que mantiene su estándar.

Lenguajes de marcas normativos

Los lenguajes de marcas normativos son aquellos que definen la estructura de otros documentos de marcado. En el contexto XML, los esquemas más comunes son DTD (Document Type Definition) y XSD (XML Schema Definition).

  • DTD: Un DTD es un documento que especifica qué elementos y atributos pueden aparecer en un documento XML, su orden y repetición, etc.

    “Un esquema DTD…define la estructura válida de un documento XML: los elementos, atributos… que pueden aparecer, el orden y el número de veces que pueden aparecer”.

    El procesador XML usa el DTD para validar un documento, comprobando que siga las reglas establecidas. Los DTD surgieron originalmente con SGML y fueron integrados en XML para la validación gramatical. Sin embargo, son relativamente sencillos y carecen de soporte para aspectos modernos (p. ej. namespaces o tipos de datos complejos); hoy suelen reemplazarse por XSD.

  • XSD (XML Schema): XSD (XML Schema) es un lenguaje de esquema basado en XML, desarrollado por el W3C, mucho más potente que DTD. Es un lenguaje de esquema utilizado para describir la estructura y las restricciones de los contenidos de los documentos XML de una forma muy precisa. A diferencia de DTD, XSD se escribe en XML (por lo que es extensible y legible por procesadores XML estándar) y permite definir tipos de datos (números, cadenas, fechas, patterns, enums, etc.), cardinalidades precisas y namespaces.

    Por ejemplo, en XSD se puede especificar que el elemento <edad> debe ser un entero entre 0 y 120, algo imposible en DTD. XSD es recomendado por el W3C desde 2001 para validar documentos XML complejos.

  • SGML: Es un metalenguaje ISO (ISO 8879:1986) del que derivan muchos otros lenguajes de marcado. SGML define una sintaxis muy rica para incluir marcas en textos y usar esquemas de validación (DTD). En su día fue la base para estandarizar documentos a gran escala, pero su complejidad lo hizo poco práctico para aplicaciones generales. De hecho, HTML surgió como una simplificación de SGML para documentos en internet. Aunque hoy en día SGML ya casi no se utiliza directamente fuera de aplicaciones industriales muy especializadas (por ejemplo, bibliotecas grandes o manuales militares), es la “raíz” histórica de XML y DocBook.

Usos comunes

Los lenguajes de marcas se usan en multitud de contextos:

  • Desarrollo Web: HTML es el pilar de cualquier página web. CSS y JavaScript actúan sobre su estructura. XML también se usa en webs (p.ej. para datos, SVG, RSS). JSON y YAML se emplean masivamente en APIs, configuraciones y bases de datos documentales.


  • Intercambio de datos: XML, JSON y YAML se usan para comunicar datos entre servicios (web services, REST APIs, archivos de configuración). JSON es muy popular en aplicaciones web y servicios modernos.


  • Documentación técnica: LaTeX y DocBook se emplean para redactar manuales, libros y artículos científicos. Permiten generar salidas profesionales (PDF, HTML, ePub) a partir de una fuente semántica.


  • Normalización y validación: DTD y XSD se usan cuando es necesario definir o validar la estructura de documentos XML (por ejemplo, archivos de configuración complejos, mensajes empresariales, formatos de datos especializados). SGML sigue vigente en entornos donde los documentos deben seguir reglas formales muy estrictas.


  • Datos geoespaciales: GML se usa en sistemas de información geográfica para intercambiar mapas y datos espaciales en proyectos de cartografía, agricultura de precisión, aplicaciones de GPS, etc. En resumen, los lenguajes de marcas son ubicuos: permiten desde construir la interfaz de un sitio web hasta modelar experimentos científicos o definir estándares de datos empresariales. Esta flexibilidad explica por qué tecnologías basadas en XML/HTML (y ahora JSON/YAML) son tan ampliamente adoptadas.

Recursos útiles

Para profundizar en estos lenguajes se recomiendan recursos oficiales y educativos de calidad:

  • W3C (World Wide Web Consortium): Especificaciones formales de HTML5, XML, XML Schema, SGML, etc. (p. ej. la especificación HTML5 en https://html.spec.whatwg.org).


  • MDN Web Docs (Mozilla): Guía completa en español para HTML, CSS y XML (por ejemplo, MDN Learn HTML).


  • W3Schools: Tutoriales prácticos y referencias rápidas (aunque sin rigor formal), útiles para ejemplos de HTML, XML y JSON.
